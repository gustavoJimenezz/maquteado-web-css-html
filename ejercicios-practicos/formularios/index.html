Formularios

Antes de empezar...

Todos los campos de nuestro form, requieren obligatoriamente el atributo name. Este campo determina el nombre bajo el cual se guardarán sus datos y si no está declarado, el campo no va a enviar nada. Obviamente, no se pueden repetir.

Es buena práctica que todos los campos tengan el atributo id (ya vamos a ver porque) el atributo id puede ser igual al atributo name, para ahorrarnos confusión. Como ya sabemos, los atributos id son únicos y no se pueden repetir

La mayoría de los campos pueden llevar un atributo value, que define el valor del mismo. Este atributo tiene diferentes funciones y formatos dependiendo del campo.

En el caso de los botones, el valor del atributo value, será también su etiqueta

En todos los casos, se puede utilizar el atributo required para indicar que un campo es obligatorio (el formulario no se enviará si no se ingresó un dato válido)

Textarea

El tag va a pedir los atributos rows y cols para definir su tamaño, pero esto es arcaico y muy dificili de medir. Es mejor utilizar CSS para determinar el tamaño

También podemos usar CSS para deshabilitar la función de modificación de tamaño (resize: none;) (Pero esto no es recomendable a menos que sea absolutamente necesario)

Textarea acepta el atributo placeholder. También se puede escribir adentro, pero esto no es recomendable (ya que obliga a usuario a borrarlo)

Los atributos maxlength y minlength se pueden usar para determinar la cantidad de caracteres habilitados

Select (y Option)

Se puede usar el atributo selected para predeterminar si una opción se ecuentra seleccionada.

Se puede usar el atributo disabled para determinar si una opción se ecuentra deshabilitada.

Podemos utilizar una combinación de estos dos atributos, junto con un value vacío y el atributo required en el select, para simular un placeholder

Se puede usar el atributo multiple para habilitar al usuario a ingresar mas de un valor.

Una vez más, podemos agregar [ ] al atributo name para agrupar respuestas múltiples

Datalist

Datalist funciona como un input de texto, pero con sugerencias.

El usuario puede escribir y aceptar una sugerencia, simplemente entrar su propia respuesta.
